# GapFinder

**역발상 투자 리서치 콘텐츠 시스템**

시장이 특정 기업을 과도하게 비관하거나 낙관하는 구간에서, 실제 사업가치와 시장 기대의 괴리를 찾아내 콘텐츠로 만드는 반자동 운영 체계입니다.

---

## 이 프로젝트를 한 마디로

> 종목 DB를 쌓고, 그 DB에서 분석 글을 뽑아내고, 이슈가 터지면 관련 종목만 골라서 업데이트하는 시스템.
> "좋은 글 계속 새로 쓰기"가 아니라, **한 번 쌓은 DB를 반복해서 재활용하는 구조**입니다.

---

## 팀 구성

| 역할 | 담당 | 하는 일 |
|---|---|---|
| **CEO** | 기획·판단·콘텐츠 방향 | 어떤 종목을 다룰지, 분석 결론이 맞는지, 글 품질 검수 |
| **CTO** | 개발·자동화·시스템 | 데이터 수집, DB 관리, 자동화 파이프라인, 코드 |
| **Claude** | 보조 | 요청에 따라 글 초안 생성, DB 조회, 문서 작성 |

---

## CEO가 알아야 할 것

### 이 프로젝트에서 CEO가 하는 일

1. **종목 요청** — "이 종목 분석해줘", "이 종목 추가해줘"
2. **글 검수** — 생성된 분석글의 숫자·결론·판단이 맞는지 확인
3. **피드백** — "이 부분 톤 수정해줘", "목표주가 근거 보완해줘"
4. **방향 결정** — "이번 주는 반도체 섹터 집중하자", "이 종목 커버 중단"
5. **진행 확인** — 현재 어디까지 왔는지, 어떤 종목이 커버되고 있는지 확인

### CEO가 안 해도 되는 일

- 코드 작성, DB 관리, 자동화 설정 — 전부 CTO가 처리
- 파일 형식이나 폴더 구조 걱정 — Claude에게 자연어로 말하면 됨

---

## 사용법 (CEO용)

이 프로젝트 폴더에서 Claude CLI를 열고, 말로 요청하면 됩니다.

### 요청 예시

```
"삼성전자 분석 글 만들어줘"
→ output/ 폴더에 분석글 초안이 생성됩니다

"현재 커버하고 있는 종목 목록 보여줘"
→ status/coverage.md 를 보여줍니다

"이번 주 진행 상황 알려줘"
→ status/progress.md 를 보여줍니다

"요청사항 남길게 — SK하이닉스 분석 추가해줘, 반도체 사이클 관점으로"
→ board/requests/ 에 요청서가 생성됩니다

"삼성전자 글 리뷰할게 — 목표주가 근거가 약해, EPS 기반으로 보완해줘"
→ board/reviews/ 에 피드백이 기록됩니다

"이번 달 반도체 섹터로 방향 잡자, 결정사항으로 남겨줘"
→ board/decisions/ 에 결정이 기록됩니다
```

## 폴더 구조

```
gap_finder/
│
├── board/                  ← CEO·CTO 소통 공간
│   ├── requests/           ← 요청사항 (CEO → CTO)
│   ├── decisions/          ← 확정된 결정사항 (둘 다)
│   └── reviews/            ← 글 리뷰·피드백 (CEO)
│
├── status/                 ← 프로젝트 현황판
│   ├── progress.md         ← 전체 진행 상황
│   ├── coverage.md         ← 종목 커버리지 현황
│   └── changelog.md        ← 변경 이력
│
├── docs/                   ← 전략 문서 (사업 설계도)
│   ├── 01_사업성 평가
│   ├── 02_플랫폼 전략
│   ├── 03_종목 선정 철학
│   ├── 04_DB 설계
│   ├── 05_매크로 업데이트 체계
│   ├── 06_로드맵
│   ├── 07_글 템플릿
│   ├── 08_리스크 통제
│   ├── 09_체크리스트
│   ├── 10_데이터 파이프라인
│   └── 11_기존 전략 비교
│
├── output/                 ← 생성된 종목 분석글
│   └── 005930_삼성전자_20260401.md
│
├── db/                     ← 종목 데이터베이스 (CTO 관리)
├── scripts/                ← 자동화 코드 (CTO 관리)
└── viz/                    ← 시각화 대시보드
```

### 각 폴더가 뭘 하는 곳인지

| 폴더 | 한 줄 설명 | 누가 쓰나 |
|---|---|---|
| **board/requests/** | "이거 해줘"를 남기는 곳 | CEO |
| **board/decisions/** | "이렇게 하기로 했다"를 남기는 곳 | CEO + CTO |
| **board/reviews/** | 글 보고 "여기 고쳐줘"를 남기는 곳 | CEO |
| **status/** | 지금 어디까지 왔는지 보는 곳 | 자동 업데이트 |
| **docs/** | 사업의 설계도 (전략·원칙·로드맵) | CEO + CTO |
| **output/** | 완성된 종목 분석 글 | Claude 생성 → CEO 검수 |
| **db/** | 종목 데이터가 저장된 곳 | CTO |
| **scripts/** | 자동화 코드 | CTO |
| **viz/** | 시각화 대시보드 | CTO |

## 핵심 철학

- "좋은 회사인가"가 아니라 **"현재 시장 기대가 과도한가"**를 분석
- 역발상(Contrarian) 기반 중기 매매 — 3~12개월 재평가 가능성 중심
- 시장의 비관/낙관이 과도한 구간에서 실제 사업가치와의 괴리를 포착

## 핵심 원칙 5조

| # | 원칙 |
|---|---|
| 1 | 신규 장문 남발 금지 — 기존 DB 재활용 + 업데이트 반복이 수익 구조 |
| 2 | 모든 글은 DB에서 출발 — 글 먼저 쓰고 정리하는 방식 금지 |
| 3 | 자동화는 재가공부터 — 목표주가 산정과 최종 결론은 자동화 금지 |
| 4 | 사람은 판단만 한다 — 데이터 정리·초안 생성·재가공은 기계 |
| 5 | 유료는 병목에만 쓴다 — '편해 보여서' 결제 금지 |
