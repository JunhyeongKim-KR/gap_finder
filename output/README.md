### CEO 피드백이 바꾼 것 (① vs ③, ② vs ④)

| 구분 | 피드백 전 (①②) | 피드백 후 (③④) |

| **transmission mechanism** | 누락 — 관세와 주가를 직접 연결 | 관세→수입물가→기대인플레→Fed→할인율→duration 자산 멀티플 |

| **결론 선명도** | "불확실성 프리미엄" / "AI capex가 변수" | "DGS10 4.50% 돌파 vs 4.00% 하회"로 행동 기준 명확 |

### 웹서치가 바꾼 것 (① vs ②, ③ vs ④)

### 결론

- **논점 프레임**(행)이 글의 분석 깊이를 결정한다. 웹서치를 아무리 해도 논점이 틀리면 "문장은 세지만 시장 독해력은 낮은" 글이 된다 (②가 이 케이스).
- **웹서치**(열)는 논점이 맞을 때 구체성·시의성·신뢰도를 끌어올린다. DB only로도 구조는 잡히지만, 최신 Fed 발언/고용 데이터/에너지 충격 같은 맥락이 빠지면 독자가 "지금 시장을 읽고 있는가?"를 의심한다.
- **최적 조합은 ④** — 올바른 논점 + 최신 데이터. 하지만 ③(DB only + 올바른 논점)도 ②(웹서치 + 잘못된 논점)보다 낫다. **논점이 먼저, 데이터는 그 다음이다.**

## 글쓰기 원칙

- **칼럼니스트형 주장문** — 리포트가 아니라 주장문으로 쓴다
- 핵심 2~4개 포인트만, 나머지는 과감히 삭제
- 결론은 문장으로 명확하게
- 상세: [docs/13_writing_principles.md](../docs/13_writing_principles.md)

## 글 생성 → 검수 흐름

```
enriched.db → 글쓰기 Agent → output/ 초안 → CEO 검수 → 발행
```

1. 글쓰기 Agent가 초안 생성 → 이 폴더에 저장
2. CEO가 검수 → `board/reviews/`에 피드백
3. 수정 반영 후 확정
4. 배포 (티스토리/네이버)
