# output — 종목 분석글

이 폴더에는 GapFinder 파이프라인 3단계(글쓰기 Agent)가 생성한 **분석글**이 저장됩니다.

## 파일 규칙

```
TICKER_종목명_YYYYMMDD.md          ← 기본
TICKER_종목명_YYYYMMDD_칼럼.md     ← 칼럼 스타일
TICKER_종목명_YYYYMMDD_리포트.md   ← 리포트 스타일 (비교용)
```

## 나스닥 관세 글 비교 (DB only vs DB + 웹서치)

| 구분 | 4/6 — DB only | 4/9 — DB + 웹서치 |
|---|---|---|
| **데이터 소스** | enriched.db (종목 분석, 매크로 민감도) | enriched.db + 실시간 웹서치 (CNBC, Tax Foundation, OANDA 등) |
| **핵심 주장** | 나스닥 하락은 실적 훼손이 아니라 불확실성 프리미엄 | 관세는 소음, 진짜 변수는 $700B AI capex의 ROI 검증 |
| **관세 분석 깊이** | 빅테크 매출 구조상 관세 직접 노출 낮다는 정성적 판단 | 대법원 IEEPA 위헌 → Section 122 전환, 15% 글로벌 관세, 의약품 100% 등 구체적 정책 변화 반영 |
| **시장 상황** | 4/6 기준 하락 국면 분석 | 4/8 나스닥 2.8% 급등(이란 휴전), "관세 내성" 형성이라는 새 해석 |
| **종목 선별** | MSFT, GOOGL, META 반등 후보 언급 | enriched.db verdict/PER/매크로 민감도 기반으로 MSFT·META 우선, TSLA·AAPL 경고 |
| **기술적 레벨** | NDX 21,000 이하 = 비관 과반영 구간 | NDX 22,500 분할 접근, 20,000~20,224 바닥 테스트 (피보나치 61.8%) |
| **전개 패턴** | A (컨센서스 반박형) | B (구조 해석형) |

**결론:** 웹서치를 결합하면 정책 변화, 시장 반응, 기술적 레벨 등 DB에 없는 실시간 맥락이 추가되어 글의 구체성과 시의성이 크게 올라간다.

## 글쓰기 원칙

- **칼럼니스트형 주장문** — 리포트가 아니라 주장문으로 쓴다
- 핵심 2~4개 포인트만, 나머지는 과감히 삭제
- 결론은 문장으로 명확하게
- 상세: [docs/13_writing_principles.md](../docs/13_writing_principles.md)

## 글 생성 → 검수 흐름

```
enriched.db → 글쓰기 Agent → output/ 초안 → CEO 검수 → 발행
```

1. 글쓰기 Agent가 초안 생성 → 이 폴더에 저장
2. CEO가 검수 → `board/reviews/`에 피드백
3. 수정 반영 후 확정
4. 배포 (티스토리/네이버)
