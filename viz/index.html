투자 철학

핵심 질문

"좋은 회사인가?" 가 아니라
"현재 시장 기대가 과도한가?"

  • 역발상(Contrarian) 기반 중기 매매
  • 3~12개월 재평가 가능성 중심
  • 시장의 비관/낙관 과도 구간 포착

투자 아이디어 7가지 유형

규제과민반응 일회성실적쇼크 업황저점 컨센서스역이용 낙폭과대 구조적경쟁우위오판 공급축소소외섹터

목표주가 방법론 4유형

  • 컨센서스 참고형 — 애널리스트 평균 TP 기준 가중
  • 절대가치평가형 — FCF / EV/EBITDA 직접 산정
  • 상대가치평가형 — 섹터 피어 멀티플 비교
  • 이벤트 리버전형 — 이슈 해소 시 복귀 기준가

3-Tier 발행 구조

  • Tier 1 — 1~2건/일, 신규 심화 분석 (장문) 사람 주도
  • Tier 2 — 3~4건/일, 기존 종목 업데이트 반자동
  • Tier 3 — 4~6건/일, 플랫폼 재가공 요약본 자동화↑

핵심 원칙 5조

1신규 장문 남발 금지 — 기존 DB 재활용 + 업데이트 반복이 수익 구조
2모든 글은 DB에서 출발 — 글 먼저 쓰고 정리하는 방식 금지
3자동화는 재가공부터 — 목표주가 산정과 최종 결론은 자동화 금지
4사람은 판단만 한다 — 데이터 정리·초안 생성·재가공은 기계
5유료는 병목에만 쓴다 — '편해 보여서' 결제 금지

22개 투자 철학 프레임워크

4개 카테고리로 분류된 투자 해석의 체계적 도구

고정 문법 패턴 (모든 글의 논리 구조)
[시장 컨센서스] → [해석의 맹점] → [실제 메커니즘] → [진짜 수혜/피해] → [체크포인트]

A. 산업/거시를 해석하는 철학 (8개)

1. 자본 사이클형

수요보다 공급을 먼저 보는 산업 분석
  • 지금 이 산업은 공급이 늘고 있는가, 줄고 있는가
  • CAPEX가 감가상각(D&A)보다 큰가
  • 신규 진입자나 IPO가 늘고 있는가
  • 지금이 사이클 상단인가, 하단인가

2. 경쟁우위/진입장벽형

초과수익이 왜 유지되는지를 보는 프레임
  • 누구나 따라 할 수 있는가
  • 고객이 쉽게 떠날 수 있는가
  • 원가 우위가 복제 가능한가
  • 규모의 경제가 있는가
  • 브랜드/전환비용/네트워크 효과가 있는가

3. 제도·인프라 설계형

기업보다 거버넌스 구조를 먼저 보는 산업 분석
  • 정부 주도형인지 민간 자율형인지
  • 참여가 의무인지 자율인지
  • 표준 API가 단일인지 복수인지
  • 허브 구조인지 직접 연계 구조인지
  • 법적으로 권리가 강한지, 실행 강제력이 있는지

4. 생태계 지속가능성형

확산 속도보다 생태계의 생존 가능성을 보는 프레임
  • 참여자들에게 경제적 유인이 있는가
  • 비용 부담이 너무 크지 않은가
  • 특정 대형 사업자만 살아남는 구조인가
  • 사업자 이탈이 발생하고 있는가
  • 고정비/과금/규제가 지나치게 무겁지 않은가

5. 데이터 거버넌스형

데이터는 기술이 아니라 권리·구조·통제의 문제
  • 데이터의 소유권은 누구에게 있는가
  • 권한 체계는 어떻게 구성되는가
  • 데이터 이동은 실시간인가, 배치인가
  • 데이터가 표준화돼 있는가
  • 허브/중계기관이 있는가
  • 플랫폼 종속(lock-in)이 있는가

6. 자원·에너지 패권형

기술의 시대 뒤에는 항상 자원과 전력의 제약이 있다
  • 누가 자원을 보유하는가
  • 누가 전력을 싸게 공급할 수 있는가
  • 누가 원자재를 가공하고 운송하는가
  • 자원 병목이 어떤 산업의 원가를 좌우하는가

7. 지정학 블록 재편형

국가 간 관계를 단기 뉴스가 아니라 구조 변화로 읽는 방식
  • 미국이 얻는 것과 잃는 것이 무엇인가
  • 유럽/동북아/중동/중앙아시아가 어떻게 다시 묶일 수 있는가
  • 한국은 어떤 블록에서 어떤 역할을 할 수 있는가

8. 병목 인프라형

병목을 쥔 업체가 진짜 수혜자
  • 시장이 주목하는 표면 테마는 무엇인가
  • 실제 병목은 무엇인가
  • 병목을 쥔 업체가 누구인가

B. 기업/종목을 해석하는 철학 (8개)

9. 기업가치 평가형

사업가치 대비 가격으로 보는 프레임
  • 이 기업의 현금창출력은 어느 정도인가
  • 현재 주가가 과도한 기대를 반영하는가
  • 가격이 너무 싸거나 비싼 이유가 무엇인가

10. 자본배분형

실적보다 돈 쓰는 방식을 보는 프레임
  • CAPEX / M&A / 배당 / 자사주 매입·소각 / 현금 보유
  • 고점에서 인수하고 저점에서 유증하는가
  • 저평가 구간에서 자사주를 사는가
  • ROIC보다 낮은 수익률의 투자에 돈을 태우는가

11. 진짜 현금흐름형

PER보다 진짜 FCF와 owner earnings를 중시
  • 유지 CAPEX와 성장 CAPEX를 구분
  • 회계상 이익과 실제 현금창출이 같은지
  • 운전자본 변화가 일시적 착시인지
  • 주식 희석이 숨어 있는지

12. 해자 지속성형

해자가 있느냐보다, 침식되고 있느냐가 중요
  • 기술 변화가 기존 해자를 무너뜨리는가
  • 규제 변화가 진입장벽을 낮추는가
  • 경영진이 해자를 강화하는 투자보다 단기 실적을 택하는가

13. 퀄리티 복리형

ROIC 지속 + 재투자 활주로가 복리의 조건
  • 높은 ROIC가 일시적인지 지속 가능한지
  • 성장할 시장이 충분한지
  • 재투자율과 성장 활주로가 있는지

14. 경영자 DNA형

오너십/인센티브/보상구조가 장기 성과를 좌우
  • 경영진이 운영자형인지 자본배분자형인지
  • 오너처럼 행동하는지
  • 자기 주식을 실제로 보유하는지
  • 보상 구조가 TSR 연동인지 EPS 장난질 유도인지

15. 데이터 인프라 수혜형

AI 자동화가 늘수록 데이터 인프라 수요 증가
  • 데이터 웨어하우스 수요
  • 쿼리 엔진 사용량
  • 권한 체계 복잡도
  • 실시간 동기화 인프라 필요성

16. 성장형 부채 해석형

부채=생산적 투자 신호일 수 있다
  • 적자 메우려고 빌린 건지
  • 데이터센터/설비/용량 증설을 위해 빌린 건지
  • RPO/백로그/수주가 늘고 있는지
  • 수익화 가시성이 높은지

C. 가격/기대 철학 (3개)

17. 기대치 역산형

주가 반영 기대치를 역산하는 프레임
  • 현재 가격이 정당화되려면 어떤 성장률/마진이 필요하나
  • 그 기대가 현실적인가
  • 시장 기대보다 조금이라도 더 좋아질 여지가 있나

18. 내러티브/역발상형

과도한 해석의 반대편을 찾는 프레임
  • 지금 시장은 무엇을 너무 당연하게 보고 있나
  • 왜 그 해석이 불완전한가
  • 반대로 해석하면 어떤 투자 아이디어가 나오는가

19. 하방 방어형

하락장 방어가 복리 핵심
  • 이 전략은 하락장에서 얼마나 버티나
  • 영구 손실 가능성이 얼마나 큰가
  • 손실 비대칭성이 얼마나 큰가

D. 사고 체계 철학 (3개)

20. 프로세스·확률형

기저율에서 시작해 확률을 업데이트하는 사고
  • 기저율을 먼저 보고
  • 새 정보로 확률을 업데이트하고
  • 항상 확률과 시나리오로 사고

21. 리스크 관리형

리스크는 변동성이 아니라 영구적 자본 손실 가능성
  • 주가 변동보다 구조적 약점을 중시
  • 과도한 기대, 과도한 레버리지 경계
  • 틀렸을 때 회복 불가능한 손실 여부

22. 멘탈모델 통합형

한 가지 프레임이 아니라 여러 관점을 통합
  • 심리 편향 / 인센티브 구조
  • 기저율 / 공급 구조
  • 네트워크 효과 / 복리

글쓰기 원칙

블로그 정체성

"시장의 기대 괴리를 찾아내는 역발상 투자 리서치 블로그"

애널리스트 리포트를 요약하지 않는다. 시장 컨센서스의 맹점을 파고들어
독자가 스스로 판단할 수 있는 체크포인트를 제공한다.

고정 문법 패턴 — 모든 글은 이 구조를 따른다
[시장 컨센서스] → [해석의 맹점] → [실제 메커니즘] → [진짜 수혜/피해] → [체크포인트]

10가지 글쓰기 원칙

1

컨센서스에서 출발하라

모든 글은 현재 시장의 지배적 의견(컨센서스)을 먼저 정리한 뒤, 그 맹점을 지적하는 구조로 시작한다.

2

주장이 아니라 메커니즘을 보여라

"오를 것이다"가 아니라 "이런 메커니즘이 작동하면 시장 기대가 수정될 것이다"로 쓴다.

3

숫자로 뒷받침하라

정성적 판단에는 반드시 정량적 근거를 붙인다. 출처와 시점을 명시한다.

4

철회 조건을 명시하라

모든 투자 아이디어에는 "이 조건이 깨지면 틀린 것"이라는 철회 조건을 반드시 포함한다.

5

시간축을 고정하라

3~12개월 중기 관점을 기본으로 하며, 글마다 투자 기간을 명시한다.

6

확신도를 등급화하라

HIGH / MEDIUM / LOW 3단계로 확신도를 표기하고, 근거를 1줄로 요약한다.

7

독자 수준을 나눠라

본문은 중급 이상 투자자 대상, 하단에 초보자용 3줄 요약을 별도 제공한다.

8

기존 글과 연결하라

동일 종목의 이전 글, 관련 산업 분석, 매크로 글을 상호 링크하여 DB를 네트워크화한다.

9

면책과 투명성을 지켜라

AI 보조 사실, 필자 포지션 유무, 정보 기준일을 매 글 하단에 명시한다.

10

사후 검증을 게시하라

3개월/6개월 후 예측 vs 실제를 비교하는 트랙레코드 글을 발행하여 신뢰를 축적한다.

Phase 0 — 설계 정리 (1~2주) 진행중

글 쓰기 전에 구조부터

  • 투자 아이디어 taxonomy 7가지 확정
  • 3-Layer 아키텍처 확정 (Agent Prompt / DB / System)
  • 투자 철학 22개 프레임워크 체계화 (A~D 4개 카테고리)
  • 글쓰기 원칙 10조 + 고정 문법 패턴 확립
  • 블로그 정체성 및 스타일 가이드 정의
  • 초기 종목 리스트 확대 (현재 2종목 → 목표 20~30개)
  • Google Sheets 4개 시트 완성
  • 글 템플릿 확정 (1개 종목 실제 작성 후 수정)

Phase 1 — 수동 검증 (1~2개월차)

정말 될지 확인하는 단계. 돈 거의 안 씀.

  • Q1: 첫 글 발행 가능한 최소 시스템 완성
  • Q2: 2종목 Tier 1 글 작성 → 품질 검증
  • KR 4~5개 + US 2~3개 종목으로 확대
  • 주 2회 발행 (화/금)
  • Tier 1 위주, 개별 종목 분석 100%
  • 성공 기준: 6~8주 내 유입 증가 추세 확인

Phase 2 — 반자동 운영 (3~4개월차)

사람이 꼭 봐야 하는 구간만 남기고 자동화

  • Q3: 데이터 파이프라인 자동화, Tier 2/3 도입
  • 종목 누적 12~15개, 산업 분석 시작
  • n8n self-hosted 도입 (VPS ~1~2만원/월)
  • 종목 60% / 산업 25% / 경쟁사 비교 15%

Phase 3 — 수익화 검증 (5~6개월차)

숫자로 확인될 때만 투자 확대

  • Q4: 수익 지표 확인 후 확장/피벗 결정
  • 월간 유입 추세 증가 확인
  • 기존 글 업데이트로 추가 조회 발생 확인
  • 고성과 주제 집중, 저성과 종목 제거

Phase 4 — 수익 최적화 (6개월 이후)

"새 글 계속 쓰기"에서 "기존 DB 활용 극대화"로

  • 실적 시즌 업데이트 자동 재소환
  • 사후 검증 글 (트랙레코드)
  • 경쟁사 교차 분석
  • 사람은 판단만 → 나머지는 DB + 자동화
