투자 철학

핵심 질문

"좋은 회사인가?" 가 아니라
"현재 시장 기대가 과도한가?"

  • 역발상(Contrarian) 기반 중기 매매
  • 3~12개월 재평가 가능성 중심
  • 시장의 비관/낙관 과도 구간 포착

투자 아이디어 7가지 유형

규제과민반응 일회성실적쇼크 업황저점 컨센서스역이용 낙폭과대 구조적경쟁우위오판 공급축소소외섹터

목표주가 방법론 4유형

  • 컨센서스 참고형 — 애널리스트 평균 TP 기준 가중
  • 절대가치평가형 — FCF / EV/EBITDA 직접 산정
  • 상대가치평가형 — 섹터 피어 멀티플 비교
  • 이벤트 리버전형 — 이슈 해소 시 복귀 기준가

3-Tier 발행 구조

  • Tier 1 — 1~2건/일, 신규 심화 분석 (장문) 사람 주도
  • Tier 2 — 3~4건/일, 기존 종목 업데이트 반자동
  • Tier 3 — 4~6건/일, 플랫폼 재가공 요약본 자동화↑

핵심 원칙 5조

1신규 장문 남발 금지 — 기존 DB 재활용 + 업데이트 반복이 수익 구조
2모든 글은 DB에서 출발 — 글 먼저 쓰고 정리하는 방식 금지
3자동화는 재가공부터 — 목표주가 산정과 최종 결론은 자동화 금지
4사람은 판단만 한다 — 데이터 정리·초안 생성·재가공은 기계
5유료는 병목에만 쓴다 — '편해 보여서' 결제 금지

  • 초보자 3줄 요약 자동 변환

stock_master (종목 마스터)

  • 기본 정보 — ticker, name, market, sector, market_cap
  • 비즈니스 — summary, revenue_structure, moat, competitors
  • 투자 분석 — thesis, thesis_type(7종), contrarian_angle, risks
  • 밸류에이션 — framework(4종), PER/PBR, hist bands
  • 목표주가 — bull/base/bear + assumptions
  • 판단 — verdict, conviction, horizon, history
  • 매크로 연동 — exposures tags, sensitivity map

financials (분기별 재무)

  • ticker, fiscal_period
  • revenue, operating_income, net_income
  • eps, bps, roe, debt_ratio
  • ocf, fcf, per, pbr, ev_ebitda
  • source, collected_at

price_daily (일별 시세)

  • ticker, date
  • open, high, low, close, volume
  • market_cap

macro_events (매크로 이벤트)

  • event_id, event_date, category
  • title, description
  • affected_tags, severity
  • action_taken

content_log (발행 이력)

  • ticker, content_type (Tier 1/2/3)
  • title, platform, url
  • published_at, needs_update

단계별 로드맵

Phase 0 — 설계 정리 (1~2주)

글 쓰기 전에 구조부터

  • Google Sheets 4개 시트 완성
  • Notion 3개 페이지 세팅 (템플릿/원칙/SOP)
  • 투자 아이디어 taxonomy 7가지 확정
  • 초기 20~30개 종목 리스트 + DB 입력
  • 글 템플릿 확정 (1개 종목 실제 작성 후 수정)

Phase 1 — 수동 검증 (1~2개월차)

정말 될지 확인하는 단계. 돈 거의 안 씀.

  • KR 4~5개 + US 2~3개 종목 발행
  • 주 2회 발행 (화/금)
  • Tier 1 위주, 개별 종목 분석 100%
  • 성공 기준: 6~8주 내 유입 증가 추세 확인

Phase 2 — 반자동 운영 (3~4개월차)

사람이 꼭 봐야 하는 구간만 남기고 자동화

  • 종목 누적 12~15개, 산업 분석 시작
  • n8n self-hosted 도입 (VPS ~1~2만원/월)
  • 종목 60% / 산업 25% / 경쟁사 비교 15%

Phase 3 — 수익화 검증 (5~6개월차)

숫자로 확인될 때만 투자 확대

  • 월간 유입 추세 증가 확인
  • 기존 글 업데이트로 추가 조회 발생 확인
  • 고성과 주제 집중, 저성과 종목 제거

Phase 4 — 수익 최적화 (6개월 이후)

"새 글 계속 쓰기"에서 "기존 DB 활용 극대화"로

  • 실적 시즌 업데이트 자동 재소환
  • 사후 검증 글 (트랙레코드)
  • 경쟁사 교차 분석
  • 사람은 판단만 → 나머지는 DB + 자동화
