## DB 저장 방식

**SQLite (단일 파일)** → 6개월 후 필요 시 PostgreSQL/Baserow

- 설치 불필요, 파일 1개 = DB 전체
- iCloud/Git으로 백업 간단
- Python 표준 라이브러리로 바로 사용
- 30종목 × 수년 데이터에서 성능 문제 없음

### 프로젝트 구조

```
gapfinder/

├── raw/                  # 원본 수집 데이터 (JSON/CSV)

```

## Workflow 구성

```
DAILY TRIGGER (cron 07:00 / 18:00)
    │
    ├── ① 데이터 수집 (Python)
    │   KRX 시세, 네이버 컨센서스, Yahoo Finance, DART 공시
    │
    ├── ② 매크로 스캔 (Python)
    │   FRED 금리, 한은 환율, 유가/원자재, Google News
    │
    ▼
③ SQLite DB 업데이트
    raw/ → 정제 → financials, price_daily 갱신
    매크로 이벤트 감지 → macro_events 등록
    │
    ▼
④ 업데이트 필요 종목 판별
    macro_sensitivity=HIGH → NEEDS_UPDATE
    실적 발표일 ±3일 → 플래그
    last_updated > 30일 → 정기 점검
    │
    ▼
[사람 확인] ← 알림 (텔레그램)
    승인 / 수정
    │
    ├── Tier 1 (Sonnet, 사람 검수 ○)
    ├── Tier 2 (Haiku, 사람 검수 ○)
    ├── Tier 3 (Haiku, 자동 발행)
    │
    ▼
⑤ 콘텐츠 생성 (LLM Agent)
    DB → 시스템 프롬프트 + 템플릿 → 글 초안
    │
    ▼
[사람 확인] ← Tier 1/2만 검수
    │
    ▼
⑥ 멀티플랫폼 발행 (n8n)
    원문 → 티스토리
    축약본 → 네이버
    1줄 요약 → X/트위터
    content_log 업데이트
```

### 사람 개입 지점 (딱 2곳)

1. **종목 업데이트 승인** — 어떤 종목을 어떤 우선순위로
2. **Tier 1/2 글 검수** — 숫자, 목표주가 가정, 철회조건, 결론
