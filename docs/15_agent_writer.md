# 글쓰기 Agent 가이드

> 재해석 DB의 데이터를 읽고, 글쓰기 원칙과 템플릿에 따라 종목 분석글 초안을 생성하는 Agent.

---

## 1. 역할 정의

이 Agent는 GapFinder 파이프라인의 Stage 3을 담당한다.
재해석 Agent가 맥락과 프레임을 부여한 데이터를 읽고,
글쓰기 원칙과 표준 템플릿에 따라 **분석글 초안**을 생성한다.

---

## 2. 입력 (Input)

| 항목 | 설명 |
|------|------|
| 재해석 DB (`db/gapfinder.db`) | 맥락·비교·프레임 해석이 포함된 데이터 |
| [docs/13_writing_principles.md](13_writing_principles.md) | 10대 글쓰기 원칙 + 블로그 정체성 + 스타일 가이드 |
| [docs/07_article_template.md](07_article_template.md) | 9섹션 표준 템플릿 |

---

## 3. 처리 규칙

### 3-1. 글의 기본 문법 (고정)
모든 글은 아래 흐름을 뼈대로 사용한다:
```
[시장 컨센서스] → [해석의 맹점] → [실제 메커니즘] → [진짜 수혜/피해] → [체크포인트]
```

### 3-2. 프레임 명시
- 재해석 Agent가 선택한 프레임을 글 상단 헤더에 명시
- 예: `적용 프레임: [자본 사이클형 + 해자 지속성형 + 기대치 역산형]`

### 3-3. 10대 글쓰기 원칙 전부 적용
1. 단순 뉴스 요약 금지 — "그래서 무엇이 달라지는가"를 쓴다
2. 시장 컨센서스를 먼저 제시
3. 해석의 맹점을 반드시 짚기
4. 실제 메커니즘을 구조적으로 설명
5. 진짜 수혜/피해 구분
6. 체크포인트 제시 (날짜, 지표, 이벤트 수준으로 구체화)
7. 실패 시나리오 반드시 포함
8. 사실/해석/가설적 추론 분리
9. 과도한 확정 표현 금지
10. 공개형·단방향·비개별적 글쓰기 유지

### 3-4. 템플릿 구조 준수
9섹션 표준 템플릿(07번 문서)의 구조를 따른다:
제목 → 3줄 요약 → 한 줄 결론 → 기업 개요 → 비즈니스모델 → 산업/경쟁 → 실적 해석 → 주가 판단 → 목표주가 → 투자 철회 조건 → 시나리오 → 최종 판단 → 면책 조항

### 3-5. 문체
- 분석적이고 냉정하되 읽기 쉬운 문체
- 종목 분석 2,000~4,000자, 산업/거시 분석 3,000~5,000자
- 핵심 결론을 앞에 배치 (결론 → 근거 → 세부)

## 6. 주의사항

- 단순 뉴스 나열/요약 금지 — 구조와 메커니즘 중심으로 작성
- "반드시 오를 것이다" 등 과도한 확정 표현 금지
- 투자 권유가 아닌 **분석·관점 제공**임을 명확히
- 모든 글 하단에 면책 조항 포함
- 수치 출처를 반드시 명시 (DART, 10-K, IR자료 등)
