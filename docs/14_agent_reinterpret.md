# 재해석 Agent 가이드

> raw DB의 날것 데이터를 읽고, 투자 철학 프레임을 적용하여 맥락·비교·해석을 부여한 후 재해석 DB에 저장하는 Agent.

---

## 1. 역할 정의

이 Agent는 GapFinder 파이프라인의 Stage 2를 담당한다.
DB에 수집된 원시 데이터(숫자, 뉴스, 매크로 지표)에 투자 철학 프레임워크를 적용하여
**맥락 있는 해석 데이터**로 변환하고, 재해석 결과를 DB에 기록한다.

---

## 2. 입력 (Input)

| 항목 | 설명 |
|------|------|
| raw DB (`db/gapfinder.db`) | 수집된 날것 숫자, 뉴스, 매크로 지표 |
| [docs/12_investment_philosophy.md](12_investment_philosophy.md) | 22개 투자 철학 프레임 (A~D 그룹) |

---

## 3. 처리 규칙

### 3-1. 프레임 선택
- 종목별로 적용 가능한 프레임을 **최소 3개** 선택
- 거시/산업 데이터에는 A 프레임(1~8) 위주, 종목 데이터에는 B+C 프레임(9~19) 위주
- 모든 해석에 D 프레임(20~22) 적용
- 프레임 간 결론이 충돌할 경우, 충돌 지점을 명시하고 판단 근거 기록

### 3-2. 맥락 부여
- **전분기 대비** (QoQ) 변화율 계산
- **전년 동기 대비** (YoY) 변화율 계산
- **컨센서스 대비** 서프라이즈/미스 판정

### 3-3. 밸류에이션 분석
- 5년 PER/PBR 밴드 내 현재 위치 계산
- 역사적 평균 대비 괴리 정도 산출

### 3-4. 정성 분석
- 뉴스 요약 + 센티먼트 판단 (긍정/중립/부정)
- 매크로 지표 → 종목 영향도 판단 (높음/중간/낮음)

### 3-5. 기록 원칙
- 적용한 프레임 번호와 선택 근거를 반드시 기록
- **사실과 해석을 구분**하여 저장 (필드 분리)

---

## 4. 출력 (Output)

| 항목 | 설명 |
|------|------|
| `stock_master` 확장 필드 | `applied_frameworks`, `roic_wacc_spread`, `moat_durability`, 센티먼트, 밴드 위치 등 |
| 재해석 결과 요약 | 종목별 1페이지 분량의 구조화된 해석 데이터 |

## 6. 주의사항

- 확인되지 않은 수치는 반드시 **가설**로 표시
- 해석에 사용한 데이터 출처를 명기
- 프레임 적용 근거 없이 결론만 기록하지 않는다
- 단일 프레임에만 의존한 해석은 금지 (최소 3개 교차 적용)
