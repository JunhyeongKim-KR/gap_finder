# 8. 운영 리스크와 통제 원칙

| 리스크 | 발생 조건 | 통제 방안 |
|---|---|---|
| **잘못된 정보** | 미확인 수치 인용, 출처 없는 데이터 | ① 작성 원칙 10조 엄수 ② 수치마다 출처 태그 필수 ③ 발행 전 팩트체크 체크리스트 |
| **최신성 부족** | 실적 발표 후 미반영, 매크로 변화 미반영 | ① earnings_calendar로 실적일 알림 ② 매크로 48시간 룰 ③ 30일 미업데이트 자동 플래그 |
| **과도한 단정** | 불확실한 상황에서 확정적 표현 | ① 사실/해석/추론 3단 구분 ② 확신도 반드시 명시 ③ 목표주가에 가정 조건 병기 |
| **자동화 품질 저하** | 멀티포스팅 포맷 깨짐, 오래된 글 재게시 | ① 자동 업로드 후 수동 검수 (첫 3개월) ② 플랫폼별 포맷 템플릿 고정 ③ 업로드 전 diff 확인 |
| **법적 리스크** | 투자 권유로 오인 | ① 매 글 면책 문구 ② "분석 의견" 표현 일관 사용 ③ 매매 타이밍 지정 자제 |
| **트랙레코드 리스크** | 틀린 판단 누적 | ① 월간 리뷰 솔직 복기 ② 판단 변경 시 이유 명시 ③ "틀린 이유 분석" 콘텐츠화 |

## 핵심 원칙 5조

| 원칙 | 내용 |
|---|---|
| 1 | 신규 장문 남발 금지 — 기존 DB 재활용 + 업데이트 반복이 돈 되는 구조 |
| 2 | 모든 글은 DB에서 출발 — 글 먼저 쓰고 정리하는 방식은 금지 |
| 3 | 자동화는 재가공부터 — 목표주가 산정과 최종 결론은 자동화 금지 |
| 4 | 사람은 판단만 한다 — 데이터 정리·초안 생성·재가공은 기계 |
| 5 | 유료는 병목에만 쓴다 — '편해 보여서' 결제 금지 |
