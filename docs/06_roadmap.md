# 6. 초기 3개월 운영 로드맵

## Phase 0 — 설계 정리 (1~2주)

글 쓰기 전에 구조부터. 이 단계를 건너뛰면 나중에 처음부터 다시 짜야 한다.

### 완료 체크리스트
1. Google Sheets 4개 시트 완성 (종목 마스터/투자 논거/밸류에이션/콘텐츠 운영)
2. Notion 3개 페이지 완성 (글 템플릿/작성 원칙/운영 SOP)
3. 투자 아이디어 유형 taxonomy 확정 (7가지)
4. 목표주가 방법론 taxonomy 확정 (4가지)
5. 초기 20~30개 종목 리스트 + 투자 아이디어 초안 DB에 입력
6. 글 템플릿 초안 완성 — 실제로 1개 종목 써보고 수정

## Phase 1 — 수동 검증 (1~2개월차)

| 항목 | 내용 |
|---|---|
| **목표** | DB 구조 확정, 플랫폼 세팅, 종목 5~8개 발행 |
| **종목 수** | KR 4~5개 + US 2~3개 |
| **콘텐츠 비중** | 개별 종목 분석 100% |
| **주간 발행** | 주 2회 (화/금) |

### 주간 운영 루틴

| 요일 | 작업 |
|---|---|
| 월요일 | 매크로 점검 — DB 종목 영향 확인, 업데이트 필요 종목 태깅 |
| 화~수 | Tier 1 신규 심화 분석 1~2건 작성, 티스토리 원문 발행 |
| 목요일 | Tier 2 기존 종목 업데이트 2~3건 |
| 금요일 | Tier 3 네이버 재가공 요약본 4~6건 |
| 주말 | 성과 확인 — 조회수/체류시간/수익 추이 분석 |

### 성공 기준
- 6~8주 내 월간 유입 증가 추세 확인
- Tier 1/2/3 중 어떤 유형이 가장 잘 먹히는지 파악
- 업데이트 글 vs 신규 글 효율 차이 확인

## Phase 2 — 반자동 운영 (3~4개월차)

| 항목 | 내용 |
|---|---|
| **목표** | 종목 누적 12~15개, 산업 분석 시작 |
| **콘텐츠 비중** | 종목 60% / 산업 25% / 경쟁사 비교 15% |

### 자동화 OK vs 금지

| n8n으로 자동화 OK | 사람이 직접 확인 (자동화 금지) |
|---|---|
| 제목 초안 생성 (LLM 3안) | 숫자 출처 확인 및 적정성 |
| 초보자 3줄 요약 자동 변환 | 목표주가 가정 적정성 |
| 티스토리→네이버 문안 분리 | 투자 논리의 내적 일관성 |
| DB 발행 상태 자동 업데이트 | 최종 결론 문장 |
| 매크로 이슈 발생 시 종목 자동 태깅 | 철회조건 정의 |

## Phase 3 — 수익화 검증 (5~6개월차)

**투자 확대 판단 기준 (3개 모두 충족 시)**
- 월간 유입이 추세적으로 증가 중
- 기존 글 업데이트만으로도 추가 조회 발생
- 사람 1시간 투입당 기대수익이 의미 있게 올라감

## Phase 4 — 수익 최적화 (6개월 이후)

비중 늘릴 콘텐츠:
- 실적 시즌 업데이트 — 기존 종목 분기마다 자동 재소환
- 매크로 이슈 반영 — 관련 종목 즉시 업데이트
- 재평가 시점 분석 — 사후 검증 글
- 경쟁사 비교 — 기존 커버 종목 간 교차 분석
