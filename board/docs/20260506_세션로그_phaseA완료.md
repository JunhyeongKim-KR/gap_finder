## 4. 핵심 컨벤션 — 다음 세션에서 지킬 것

### 4-1. 코딩 컨벤션
- Python 3.11+, type hint 사용
- Anthropic SDK는 `claude-opus-4-7` (2단계만), 나머지는 `claude-sonnet-4-6`
- 환경변수: `python-dotenv` 로드, `.env` 사용
- 외부 API 호출은 `crawler/<service>.py` 모듈 통해서만 (직접 호출 금지)

### 4-2. 문서 컨벤션
- 각 단계 prompt 변경 시 `pipeline/<stage>/CHANGELOG.md` 갱신 필수
- 버전: v<major>.<minor>.<patch>
- 새 prompt 버전은 `pipeline/<stage>/versions/v<X>.md` 보관 권장 (현재 active는 prompt.md)

### 4-3. board/ 컨벤션
- 요청: `board/requests/YYYYMMDD_제목.md` (frontmatter: 요청자/날짜/상태)
- 결정: `board/decisions/YYYYMMDD_제목.md`
- 리뷰: `board/reviews/YYYYMMDD_종목명_리뷰.md`
- 확정 전략: `board/docs/`

### 4-4. 커밋 컨벤션
- `[CEO|CTO|Claude] type: msg` 형식
- type: feat/fix/docs/refactor/chore
- 예: `[CTO] feat: Phase A 인프라 구축 — ...`
- conventional commits 강요 X (Dnotitia 룰 영향, GapFinder는 자유롭지만 위 패턴 유지)

### 4-5. 슬래시 커맨드 컨벤션
- 자연어 매칭 키워드는 `CLAUDE.md` §"슬래시 커맨드 ↔ 자연어 트리거" 표가 진실
- 콘텐츠 생성·발행 흐름은 슬래시 우회 금지 (운영 데이터 누락 방지)
- 자연어 → 슬래시 자동 invoke. CEO에게 "다음 단계 갈까요?" 묻지 않음.
- 예외: stage4 draft → stage5 polish는 명시 OK 필요
