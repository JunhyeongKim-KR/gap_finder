## 4. 구축 순서 (CEO 보고서 A~F 단계 매핑)

CEO 보고서의 **B단계(2단계 prompt 검증)가 최우선 unknown**. 인프라(A)와 병렬 추진.

### Phase A — 인프라 셋업 (1~2주)
- [ ] Langfuse self-host 배포 (Docker Compose)
- [ ] Anthropic API 키 환경 분리 (dev / prod)
- [ ] 프롬프트 폴더 구조 생성 (`/prompts/stage[1-5]_*/v1.0.md`)
- [ ] 평가셋 폴더 구조 생성 (`/eval_sets/stage[1-5]/`)
- [ ] 실행 스크립트 골격 (`agents/pipeline.py`)
- [ ] CHANGELOG 컨벤션 문서화

### Phase B — 2단계(분석 관점) 우선 검증 ⭐ (최우선)
**전체 프로젝트 의미를 좌우. 다른 단계 만들기 전 반드시 통과.**

- [ ] `prompts/stage2_angle/v1.0.md` 초안 작성 (시장 통념 반박 thesis 3개 생성)
- [ ] `eval_sets/stage2/sample_001~005.yaml` 5개 작성
  - 오라클 부채 우려 / 에코프로 어닝쇼크 / SK하이닉스 RPO / 삼바 FDA / TSMC capex
- [ ] LLM-as-judge prompt `prompts/judge/v1.0.md` 작성
- [ ] 5종목 실제 시험 → CEO 시연
- [ ] **CEO go/no-go 결정 후 다음 단계 진입**
- [ ] no-go 시: 프롬프트 개선 또는 전체 plan 재검토

### Phase C — 나머지 4단계 구축 (B 통과 후, 2~3주)
- [ ] 1단계 글감 잡기 (prompt + eval set)
- [ ] 3단계 근거 수집 (DB 조회 + 외부 API 통합)
- [ ] 4단계 글 작성 (`docs/07_article_template.md` 활용)
- [ ] 5단계 다듬기 + 후킹 강도 3변형
- [ ] 5단계 chain end-to-end 검증 (1종목 발행 직전까지)

### Phase D — CEO 인터페이스 (Phase C와 병렬 가능)
- [ ] Streamlit 대시보드 골격
- [ ] 후보 카드 UI (3개 비교 + 1-click 픽 + 코멘트)
- [ ] 단계별 진행 상황 표시
- [ ] CEO 코멘트 자동 inject 검증

### Phase E — 발행 자동화 (D 완료 후)
- [ ] 매주 월요일 글감 후보 15개 자동 산출 (`agents/longlist.py`)
- [ ] CEO 픽 → 5단계 자동 진행 → 티스토리 발행
- [ ] 운영 데이터 누적 (`db/operations.db` — pick / reject / comment)
- [ ] 거부·재생성 횟수 한도 3회 enforce

### Phase F — 모니터링
- [ ] Search Console + GA4 연동 (자동 데이터 수집)
- [ ] 글당 토큰 비용 자동 표시
- [ ] 운영 데이터 dashboard (CEO 픽 비율 / 수정 비율 / 1편 작업 시간 / 사실 오류율)
- [ ] 발행 30편 시점 PV·체류시간·CTR·재방문률 측정 시작
