# 1. 사업성 평가

## 왜 먹힐 수 있는가

- **공급 부족 영역**: 한국 투자 콘텐츠 시장에서 "역발상 관점 + 명확한 판단 + 체계적 업데이트"를 동시에 제공하는 채널은 거의 없음. 대부분 순방향(좋은 회사 찾기) 또는 차트 기반.
- **검색 수요 상시 존재**: "OO 주가 전망", "OO 목표주가"는 연중 검색량이 꾸준한 상록수 키워드. 종목 수가 늘수록 유입 채널이 기하급수적으로 증가.
- **업데이트 구조가 곧 해자**: 매크로 이슈 발생 시 기존 글을 업데이트하면 콘텐츠 신선도 유지 + 재방문 유도 + SEO 가점. 일반 블로거는 이걸 수동으로 못 함.
- **수익화 경로 다변화 가능**: 블로그 광고 → 유료 뉴스레터 → 유료 DB 접근 → 강의/출판으로 확장 가능.

## 한계

- **1인 운영의 병목**: 종목 수 증가 시 DB 유지보수 부담 급증. 초기 30종목 이내로 제한 필요.
- **트랙레코드 리스크**: 틀린 판단이 누적되면 신뢰 붕괴. 반드시 과거 판단 결과를 투명 공개하고, 틀린 건 왜 틀렸는지 복기 콘텐츠화해야 함.
- **법적 경계**: 목표주가/손절가 제시는 투자 권유로 오인될 수 있음. 면책 문구 + "분석 의견이지 투자 권유가 아님" 명시 필수.
- **데이터 접근**: 한국주식은 FnGuide/DART, 미국주식은 SEC/Yahoo Finance 등에서 수동 수집이 초기에는 불가피. 자동화 전까지 시간 소모 큼.

## 성공 핵심 요소

| 요소 | 설명 |
|---|---|
| DB 최신성 | 분기 실적 발표 후 72시간 내 업데이트 |
| 판단 명확성 | 매 글 "매수/관망/회피" 중 하나로 결론 |
| 트랙레코드 투명성 | 월 1회 과거 판단 리뷰 콘텐츠 |
| 매크로 연동 속도 | 이슈 발생 48시간 내 관련 종목 업데이트 |
| 초기 종목 집중 | 15~20개로 시작, 깊이 우선 |
