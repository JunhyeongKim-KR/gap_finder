# 데이터 소스 크롤링 우선순위 정리

## CEO 핵심 지시
> "API로 붙일 수 있는 공식 소스는 최대한 API로 붙이고, 정책 본문·IR 본문·뉴스 해석은 공식 웹 크롤링으로 보완하자. 스타일 학습용 크롤링은 글 생성용 배경지식 크롤링과 별도 파이프라인으로 분리하자."

## 소스 분류 (4단계)
1. **공식 공개 API** — 가장 안전, 최우선
2. **공식 사이트 크롤링** — API 없는 정책/IR 본문
3. **민간 사이트 크롤링** — 보조재 (팩트 확정은 공식으로)
4. **로그인/유료** — 후순위 또는 제외

## 1차 구축 (즉시)
- OpenDART (공시, 재무, 유증/배당/지분)
- KRX Open API (일별 시세, 지수, 매매정보)
- ECOS (금리, 환율, 물가, 국민계정)
- 관세청 무역통계 API (국가별 수출입)
- SEC EDGAR API (10-K, 10-Q, XBRL)
- FRED (금리, 유동성, 경기, 신용스프레드)
- EIA (원유, 가스, 발전, 전력가격)
- BLS (고용, 실업, 임금, 물가)
- 금융위/산업부/관세청 보도자료 (크롤링)
- 기업 IR/실적 발표 (크롤링)

## 2차 구축
- KIND, 금감원, KDI, 자본시장연구원
- Reuters, Bloomberg, FT, WSJ
- 네이버증권, 한경, 매경, 산업 전문지

## 3차 구축
- 스타일 학습용 블로그/서브스택
- 대체 데이터 (선박운임, 전력사용량, 반도체 장비 출하 등)
- 채용/특허/논문

## 운영 원칙
- robots.txt 확인, rate limit 준수
- SEC: 10 req/sec 제한
- 원문 저장 vs 재배포 구분
- 기사 본문은 요약/특징값 추출 중심

## CTO 대응
- DB를 raw.db / enriched.db 2개로 분리
- 기존 gapfinder.db 제거, 새로 시작
- 1차 구축 소스부터 크롤링 스크립트 구현
